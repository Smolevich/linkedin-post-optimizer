# Voice Guidelines & AI Slop Detection

## AI Language Patterns to Remove

Before finalizing any post, check for and remove these tells:

- "In today's fast-paced world..."
- "It's important to note that..."
- "Let's dive in..."
- "Here's the thing..."
- "Game-changer", "unlock", "leverage", "synergy"
- Excessive hedging ("I think", "in my opinion", "perhaps")
- Perfect parallel structure in every list
- Overly smooth transitions

## Formatting Red Flags

- Too many emoji (max 1, prefer 0)
- Perfect bullet point symmetry
- Headers in a short post
- Hashtag spam (max 3-5, only if natural)

## Voice Check

Ask: "Would this person actually say this out loud?" If no — rewrite in simpler, more natural language.

The best posts feel slightly raw, like a first draft that was cleaned up — not polished to perfection.

## Anti-Patterns

**Never do:**
- Add things the user didn't mention (don't invent achievements)
- Use words the user wouldn't use
- Make it longer than needed
- Add CTAs if engagement wasn't requested
- Force a "viral" structure on a simple thought
- Use emoji the user wouldn't use
- Add hashtags if the user doesn't normally use them

**Always do:**
- Preserve the user's actual voice
- Keep specific details and numbers from the original
- Match the user's usual post length
- Respect style preferences
- Suggest, don't dictate

## Toxic Tone Anti-Pattern

LinkedIn is a professional network. Even when reviewing a bad product, maintain a constructive tone. Never generate posts with:

- Crude language or profanity ("засрал проект", "дерьмовые практики")
- Contemptuous dismissal of tools or companies ("он помер")
- Passive-aggressive sarcasm ("видимо через годик добавят")
- Punching down at specific products without constructive critique

**Bad (toxic):**
"Провел тест Junie от JetBrains. За вечер засрал чистый проект до уровня энтерпрайз легаси со смесью самых дерьмовых практик. Потом он помер, и через два часа переустановок я увидел что-то про лимиты. Баланс видимо через годик добавят)"

**Good (critical but professional):**
"Протестировал Junie от JetBrains на чистом проекте. За вечер кодовая база превратилась в типичный энтерпрайз: смешанные паттерны, дублирование, нет единой архитектуры. Потом агент остановился — оказалось, лимиты. Информация о балансе пока спрятана в консольных логах, в UI её нет.

Инструмент сырой. Потенциал есть, но пока не для продакшн-задач."

The difference: same honest assessment, no contempt. Critique the product, not mock it.

## Voice Preservation Example

**User writes:** "spent whole weekend debugging this stupid auth bug, finally fixed it by reading the actual docs lol"

**Bad (AI slop):**
"This weekend, I encountered a challenging authentication bug. After extensive debugging, I discovered the solution was in the documentation all along. Key lesson: always RTFM! #Engineering #Debugging"

**Good (preserved voice):**
"Spent the whole weekend on one auth bug. The fix? Actually reading the docs.

Sometimes the answer is embarrassingly obvious."
